## Описание проекта
**NotesApp** — это ASP.NET Core Web API для управления заметками и категориями. Приложение позволяет создавать категории (Работа, Учёба, Личное), добавлять в них заметки, задавать приоритеты, закреплять важные записки и архивировать старые.

Реализован полноценный CRUD с использованием следующих технологий:
- **ASP.NET Core** — веб-фреймворк.
- **Entity Framework Core** — работа с базой данных.
- **SQLite** — файловая база данных.
- **Swagger** — документация и тестирование API.

Приложение использует паттерн Repository для разделения логики, валидирует данные через Data Annotations и возвращает ответы в едином формате.

## Описание паттерна Repository

**Repository (Репозиторий)** — это паттерн проектирования, который изолирует логику доступа к данным от логики контроллера.

**В нашем приложении схема выглядит так:**
`Controller` → `Repository` → `DbContext` → `Database`

**Зачем он нужен:**
1. **Разделение ответственности:** Контроллер занимается только приемом запроса и возвратом ответа. Вся сложная логика (SQL-запросы, фильтры, сортировка) скрыта в репозитории.
2. **Тестируемость:** Контроллер можно тестировать отдельно от базы данных, подменив реальный репозиторий на тестовый (Mock).
3. **Гибкость:** Если в будущем мы захотим заменить SQLite на PostgreSQL, нам не придется переписывать код контроллеров. Нужно будет только изменить реализацию репозитория.

## Главные выводы
1. Паттерн Repository — не лишняя абстракция, а способ держать код управляемым при росте проекта.
2. Data Annotations решают двойную задачу: валидируют входные данные и задают ограничения в БД.
3. Единый формат ответа (`ApiResponse`) упрощает жизнь фронтенду — он всегда знает, что ждать от сервера.
4. `DeleteBehavior.Restrict` защищает данные от случайного каскадного удаления.
5. `Include()` и проекция в DTO через `Select()` — правильный способ получить связанные данные без N+1 проблемы.
